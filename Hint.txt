1. نمای کلی پروژه
پروژه PhoenixTrade یک پلتفرم آموزشی برای بازار طلا (فارکس) است که از سه بخش اصلی تشکیل شده: ربات تلگرام، WebApp و پنل مدیریت. هدف اصلی آموزش گام‌به‌گام مفاهیم معامله‌گری با رویکرد روانشناسی و جامعه‌شناسی مخاطب ایرانی است.

6. مدل‌های داده

brokers: Broker(id, name, logo, description, pros, cons, register_link, min_deposit, is_active, created_at)

events: Registration(id, user(FK), event(FK), registered_at, attended) با unique_together

7. APIهای ایجاد شده (Django REST Framework)
GET  /api/categories/ - لیست دسته‌بندی درس‌ها
GET  /api/lessons/ - لیست درس‌ها (فیلتر: category, is_free)
GET  /api/lessons/{id}/ - جزئیات یک درس
GET  /api/brokers/ - لیست بروکرهای فعال
GET  /api/brokers/{id}/ - جزئیات یک بروکر
GET  /api/events/ - رویدادهای آینده
GET  /api/events/{id}/ - جزئیات یک رویداد
POST /api/events/{id}/register/ - ثبت‌نام در رویداد (نیاز به احراز هویت)

9. پکیج‌های بکاپ
- django-dbbackup: برای بکاپ دیتابیس (با cron قابل تنظیم)
- djbackup: برای بکاپ فایل‌ها و کدها از طریق پنل گرافیکی در آدرس /dj-backup/

مشکل فعلی: منوی کشویی Store in در پنل djbackup گزینه LOCAL را نمایش نمی‌دهد با اینکه در settings.py تعریف شده.

16. دستورات کاربردی
وضعیت سرویس: sudo systemctl status phoenix-django
ری‌استارت: sudo systemctl restart phoenix-django
لاگ: sudo journalctl -u phoenix-django -n 50
شل جنگو: cd /var/www/phoenixtrade/django_backend && python manage.py shell
migrate: python manage.py migrate
بکاپ دیتابیس: python manage.py dbbackup
لیست بکاپ‌ها: python manage.py listbackups
